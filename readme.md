# ImageProcessor - 精细化需求文档

## 一、产品定位

一款面向设计师和开发者的桌面图像处理工具，核心定位是**批量处理**和**游戏开发辅助**（精灵图生成）。

---

## 二、功能需求（按优先级分层）

### 🔴 MVP 阶段（最小可用产品）

#### 1. 智能抠图（AI 去背景）

| 需求项     | 优先级 | 详细描述                                                         |
| ---------- | ------ | ---------------------------------------------------------------- |
| 单图抠图   | P0     | 上传单张图片，一键去除背景                                       |
| 批量抠图   | P0     | 支持多文件拖拽，批量处理并保存                                   |
| rembg 集成 | P0     | 使用 rembg 库，支持 u2net/u2net_human_seg/u2net_cloth_seg 等模型 |
| 格式支持   | P0     | 输入：JPG/PNG/WebP；输出：PNG（保留透明）                        |
| 进度显示   | P1     | 批量处理时显示进度条和预计剩余时间                               |

**细化交互流程：**

```
用户拖拽图片到软件窗口
    ↓
自动识别主体并去除背景（rembg）
    ↓
展示结果预览（棋盘格背景）
    ↓
提供「保存」或「继续编辑」选项
```

#### 2. 透明背景预览

| 需求项         | 优先级 | 详细描述                                |
| -------------- | ------ | --------------------------------------- |
| 棋盘格背景     | P0     | 经典的灰白交替棋盘格，默认 16x16px 格子 |
| 背景切换       | P1     | 支持切换：棋盘格/白色/黑色/自定义颜色   |
| 棋盘格密度调节 | P2     | 可调节格子大小（8px/16px/32px）         |

#### 3. 基础缩放与导出

| 需求项     | 优先级 | 详细描述                                             |
| ---------- | ------ | ---------------------------------------------------- |
| 等比缩放   | P0     | 按宽度或高度等比缩放，自动计算另一维度               |
| 批量缩放   | P0     | 支持多张图片统一尺寸批量处理                         |
| 预设尺寸   | P1     | 提供常用预设：50%、25%、社交媒体尺寸（1080x1080 等） |
| 多格式导出 | P0     | PNG/JPG/WebP，JPG 支持质量调节（1-100）              |
| 插值算法   | P1     | 可选：LANCZOS（高质量，默认）、BILINEAR、NEAREST     |

### 🟠 高优先级（核心功能完善）

#### 4. 精灵图（Sprite Sheet）生成器

| 需求项        | 优先级 | 详细描述                                             |
| ------------- | ------ | ---------------------------------------------------- |
| 网格布局      | P1     | 自定义网格（如 4x4、8x8、自动计算最佳布局）          |
| 间距控制      | P1     | 设置 Padding（内间距）和 Margin（外边距）            |
| 排序方式      | P2     | 按文件名、上传顺序、或自定义拖拽排序                 |
| JSON 配置导出 | P1     | 生成 TexturePacker 兼容的 JSON，包含每帧的坐标和尺寸 |
| 预览模式      | P1     | 实时预览精灵图布局效果                               |
| 边框优化      | P2     | 自动去除单图透明边缘，节省空间                       |

**JSON 输出格式示例：**

```json
{
  "frames": {
    "frame_001.png": {
      "frame": { "x": 0, "y": 0, "w": 64, "h": 64 },
      "rotated": false,
      "trimmed": false,
      "sourceSize": { "w": 64, "h": 64 }
    }
  },
  "meta": {
    "version": "1.0",
    "size": { "w": 256, "h": 256 }
  }
}
```

### 3.2 主题与配色

- **主色调**：跟随系统深色/浅色模式，或使用现代中性配色
- **强调色**：蓝色系（#3B82F6）用于主要操作按钮
- **背景色**：
  - 深色模式：#1F2937（主背景）、#111827（侧边栏）
  - 浅色模式：#F9FAFB（主背景）、#F3F4F6（侧边栏）
- **棋盘格**：#FFFFFF 和 #E5E7EB 交替

### 3.3 交互细节

- **拖拽文件**：支持拖拽到窗口任意位置上传
- **实时反馈**：操作后立即显示视觉反馈（如按钮按下效果）
- **加载状态**：耗时操作显示进度条和取消按钮
- **错误提示**：使用非阻塞式 Toast 通知，3 秒自动消失

## 四、技术架构

### 4.1 核心依赖

```
PySide6        → GUI 框架（Qt for Python）
Pillow (PIL)   → 基础图像处理
rembg          → AI 抠图
opencv-python  → 高级图像处理（可选，用于内容感知擦除）
numpy          → 数值计算
```

### 4.3 数据处理流程

```
用户操作
   ↓
GUI 层接收事件
   ↓
调用 Core 层处理方法
   ↓

   ↓
更新 ImageItem 数据
   ↓
触发 Canvas 重绘
   ↓
显示处理结果
```

## 五、性能考虑

| 场景       | 策略                                    |
| ---------- | --------------------------------------- |
| 大图片处理 | 处理时生成缩略图，避免内存溢出          |
| 批量处理   | 使用线程池（QThreadPool），不阻塞 UI    |
| AI 抠图    | rembg 首次加载模型较慢，显示加载提示    |
| 画布缩放   | 使用 QGraphicsView 的缩放功能，保持流畅 |
| 历史记录   | 限制历史栈大小，大图使用内存优化策略    |

## 七、待决策事项

1. **是否支持 OpenCV？** 如需内容感知擦除，需要 OpenCV；如只做基础功能，可省略
2. **是否打包为独立应用？** 使用 PyInstaller 打包为 .exe/.app，方便分发
3. **是否支持插件扩展？** MVP 阶段不建议，后续可考虑
4. **画布实现方式？** QGraphicsView（推荐，功能丰富）vs QLabel（简单，功能有限）
